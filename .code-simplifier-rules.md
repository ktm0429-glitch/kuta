# コード簡素化ルール

このプロジェクトでは、以下のルールに従ってコードを常に最適化します。

## 原則
- **機能は一切変更しない** - 動作は完全に保持
- **トークン消費の最小化** - AI修正時のコスト削減を最優先
- **可読性の維持** - 簡潔さと理解しやすさのバランス

## 削除対象
1. **冗長な処理**
   - 同じ処理の繰り返し
   - 不要な変数代入
   - 使用されていない関数/変数

2. **重複コード**
   - コピペされた同一ロジック
   - 似た処理は統合・共通化

3. **不要なコメント**
   - コードを読めば明らかな説明
   - 古い/間違ったコメント
   - TODOで実装済みのもの
   - ただし、**複雑なロジックの意図説明は残す**

## 構造の単純化
1. **ネストの削減**
   - Early returnの活用
   - 条件式の簡略化

2. **関数の分割**
   - 長い関数は責任ごとに分割
   - ただし過度な細分化は避ける

3. **命名の改善**
   - 短く明確な名前
   - 略語は一般的なもののみ

## 適用タイミング
- 新規コード追加時
- コード修正時
- リファクタリング時
- プルリクエスト作成前
